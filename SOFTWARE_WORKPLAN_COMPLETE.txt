This document is the complete project workplan showing:

====================================================================
A) PROJECT VISION
====================================================================
Build and scale an internal operations software for product lifecycle management:
- Product master data
- Inventory tracking
- Job/workflow management
- Workforce and KYC handling
- Draft save/resume flows
- Manager dashboards and operations visibility

Short-term architecture:
- Next.js full-stack app (frontend + internal API routes)

Long-term architecture:
- Dedicated Django + DRF backend with PostgreSQL, Redis/Celery, and production-grade APIs

====================================================================
E) WHAT WILL BE DONE NEXT (FORWARD ROADMAP)
====================================================================

PHASE 1 — Backend Bootstrap (Django Foundation)
1. Create backend directory and Python environment.
2. Initialize Django project structure and settings split (base/dev/prod).
3. Install and configure DRF, JWT, CORS, environment management.
4. Connect PostgreSQL and run baseline migrations.
5. Add healthcheck endpoint and baseline app settings.

Deliverables:
- Runnable Django backend with PostgreSQL connectivity.

PHASE 2 — Core Domain Modules (Products, Jobs, Inventory)
1. Build products app models + serializers + APIs.
2. Build jobs app with status transition logic.
3. Build inventory ledger model with atomic stock operations.
4. Introduce service layer for business logic (not in views).
5. Add role-aware permissions in APIs.

Deliverables:
- Stable core operational APIs for products/jobs/inventory.

PHASE 3 — Workforce, KYC, Drafts
1. Implement workforce module endpoints.
2. Implement KYC workflow states (pending/approved/rejected).
3. Implement drafts save/resume APIs.
4. Add validation and audit-friendly update paths.

Deliverables:
- End-to-end operational coverage for workforce + KYC + drafts.

PHASE 4 — Auth Upgrade and Security Hardening
1. Replace/finalize fixed-user approach with scalable user and role model.
2. Move to JWT-based auth flow with refresh logic.
3. Add stronger security baseline (CORS/CSRF/cookie/session policy).
4. Add login rate-limit or brute-force protection.

Deliverables:
- Production-grade identity and access model.

PHASE 5 — Frontend-to-Django Integration
1. Define API base URL strategy per environment.
2. Replace current internal API dependencies where needed.
3. Integrate token/session refresh behavior in frontend flows.
4. Validate all key pages against new backend APIs.

Deliverables:
- Frontend connected to Django APIs for production flow.

PHASE 6 — Async, Reliability, and Automation
1. Add Redis + Celery + celery-beat.
2. Move heavy operations to background tasks.
3. Add retry policies and error handling patterns.
4. Add structured logs and basic monitoring hooks.

Deliverables:
- More reliable and scalable background processing.

PHASE 7 — Testing, Documentation, and CI/CD
1. Add unit tests for service layer.
2. Add API integration tests.
3. Publish OpenAPI/Swagger docs via drf-spectacular.
4. Add lint/format/test workflow in CI.
5. Dockerize services and define deployment runbook.

Deliverables:
- Deployment-ready backend pipeline with quality gates.

====================================================================
H) RISKS AND MITIGATION
====================================================================
Risk 1: Migration complexity from Next.js APIs to Django APIs
Mitigation:
- Phase-wise endpoint migration with compatibility layer and regression checklist.

Risk 2: Data consistency for inventory/job transitions
Mitigation:
- Service-layer transactions + test cases for edge/race conditions.

Risk 3: Auth/session transition issues
Mitigation:
- Plan gradual rollout and dual validation during migration period.

Risk 4: Delays due to missing env/deployment discipline
Mitigation:
- Early .env templates, Docker setup, and CI checks.

====================================================================
I) DEFINITION OF "PROJECT READY"
====================================================================
Project is considered ready for production-scale use when:
- Django backend is primary system of record.
- PostgreSQL-backed modules cover products/jobs/inventory/workforce/KYC/drafts.
- Auth and permissions are role-based and secure.
- Frontend is integrated with stable API contracts.
- Test suite + CI/CD pipeline are active.
- Monitoring/logging and rollback runbook are available.
