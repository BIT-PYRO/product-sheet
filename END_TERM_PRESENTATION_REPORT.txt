1. INTRODUCTION
================================================================================

Product Sheet Design is a comprehensive full-stack internal business management
application developed for a jewellery manufacturing and retail operation. The
system streamlines complex workflows across product cataloguing, inventory
tracking, order management, workforce administration, customer KYC 
(Know Your Customer), and job-sheet workflows — all integrated into a unified
platform with a single deployed backend and database.

The application is built on modern, production-ready technologies and is
currently deployed and operational at:
Link: https://product-sheet-frontend.onrender.com/

The system serves as the backbone for operational excellence, enabling
real-time visibility, efficient resource allocation, and comprehensive 
business intelligence across all functional areas.

2. TOOLS AND TECH STACK USED
================================================================================

FRONTEND LAYER:
───────────────
• Framework: Next.js 16 (React 19)
  - Server and client-side rendering capabilities
  - File-based routing system
  - Built-in API routes for backend proxy
  
• Language: TypeScript / JavaScript (JSX)
  - Type-safe component development
  - Enhanced IDE support and code reliability
  
• Styling & UI:
  - Tailwind CSS v3: Utility-first responsive design
  - Radix UI + shadcn/ui: Accessible headless UI components
  - Material-UI (MUI): Additional component library for dashboard sections
  - Lucide React: Consistent, modern icon set
  
• Data & Visualization:
  - react-hook-form: Efficient form state management
  - Recharts: Interactive charts and data visualizations
  - xlsx (SheetJS): XLSX/CSV file parsing for bulk data uploads
  - pdf-parse: PDF extraction for picklist processing
  - Zod: Runtime type validation
  
• Infrastructure:
  - pnpm: Fast, disk-efficient package management
  - Next.js API routes: Proxy layer for JWT token handling
  
BACKEND LAYER:
──────────────
• Framework: Django 5 + Django REST Framework (DRF)
  - RESTful API architecture
  - Comprehensive ORM for database operations
  - Admin interface for data management
  
• Language: Python 3.11
  - High-level, readable code
  - Robust standard library
  - Extensive ecosystem for data processing
  
• Authentication & Security:
  - djangorestframework-simplejwt: JWT token authentication
  - Bearer token-based API security
  - CORS handling via django-cors-headers
  
• Database:
  - PostgreSQL: Relational data store
  - Django ORM: Query abstraction and migrations
  - Connection pooling for performance
  
• Async & Background Tasks:
  - Celery: Asynchronous task queue
  - Redis: Message broker and caching
  - Scheduled background job support
  
• API Documentation & Tools:
  - drf-spectacular: Auto-generated OpenAPI/Swagger documentation
  
• Deployment:
  - Gunicorn: WSGI production server
  - WhiteNoise: Static file serving in production
  - Render.com: Cloud hosting platform
  
BACKEND MODULES (9 Django Apps):
────────────────────────────────
1. accounts       - Custom user model and JWT authentication
2. products      - Product catalogue management
3. jobs          - Job creation, assignment, and voucher workflows
4. orders        - Order management and fulfillment
5. inventory     - Stock transactions, picklists, and findings
6. workforce     - Employee records, attendance, and HR operations
7. customers     - B2B customer data and KYC tracking
8. kyc           - Customer KYC verification workflows
9. drafts        - Form draft storage and submission
10. accounting   - Financial tracking and reporting
11. hr           - HR-specific operations (attendance, payroll, expenses)
12. calendar_integration - Google Calendar sync
13. common       - Shared utilities (timestamps, audit trails)
