PRODUCT-SHEET-DESIGN: COMPLETE TECH STACK (CURRENT + PLANNED)
Date: 25-Feb-2026

============================================================
1) FRONTEND STACK (CURRENTLY USED)
============================================================
- Framework: Next.js 16.1.6 (App Router, React Server Components enabled)
- UI Library: React 19 + React DOM 19
- Language Setup:
  - Main codebase is mostly JavaScript/JSX
  - TypeScript tooling is enabled (tsconfig present, strict mode enabled)
- Styling:
  - Tailwind CSS 3.4.17
  - PostCSS
  - tailwindcss-animate
  - CSS variables based theming
- Component System:
  - shadcn/ui configuration present (components.json)

  - Class utilities: clsx + tailwind-merge
- Forms and Validation:

  - zod
- Charts / Data UI:

- UX helpers:
  - sonner (toasts)
  - cmdk
  - embla-carousel-react
  - vaul
  - input-otp
  - react-resizable-panels

============================================================
2) FRONTEND ROUTING + MIDDLEWARE (CURRENTLY USED)
============================================================
- Next.js rewrites in next.config.mjs:
  - /api/* -> /frontend/api/*
  - multiple app routes rewritten to /frontend/* pages
- Middleware:
  - next/server middleware for auth gatekeeping
  - Protects private routes and redirects unauthenticated users to /login
  - Skips static/public asset paths

============================================================
3) BACKEND / API LAYER (CURRENTLY USED)
============================================================
- Runtime for API endpoints: Next.js Route Handlers (Node environment)
- API structure:
  - app/frontend/api/auth/login/route.js
  - app/frontend/api/auth/logout/route.js
  - app/frontend/api/auth/session/route.js
  - app/frontend/api/save-to-sheets/route.js
- Auth model currently implemented:
  - Cookie-based session (httpOnly cookie: psd-session)
  - Fixed user list in frontend/lib/auth.js
  - Session validation via middleware + session endpoint
- Data layer currently implemented:
  - No local SQL/NoSQL database configured yet in this repo
  - Primary data operations for product data are handled via Google Sheets API

============================================================
4) THIRD-PARTY INTEGRATIONS (CURRENTLY USED)
============================================================
- Google APIs (googleapis package)
- Google Sheets API v4:
  - Read/write product sheet data
  - Uses service-account style credentials from environment variables
  - Spreadsheet ID and credential fields consumed from process.env
